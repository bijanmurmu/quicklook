# Security Policy

## Supported Versions

Currently, only the latest version on the `main` branch is actively supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security and privacy of QuickLook very seriously. QuickLook is designed with a **privacy-first, zero-backend** architecture to ensure user data never leaves the browser. 

If you discover a security vulnerability within QuickLook, please DO NOT open a public issue. Instead, please report it privately:

1. Send an email to the repository maintainer (or open a private security advisory via GitHub).
2. Include steps to reproduce the vulnerability.
3. We will acknowledge receipt of your vulnerability report within 48 hours and strive to send you regular updates about our progress.
4. If a vulnerability is confirmed, we will issue a patch as quickly as possible.

Thank you for helping keep QuickLook and its users safe!
